# StockTracker — Accessibility (a11y) Audit

**Date:** 2026-07-04
**Scope:** All 10 routes and ~50 UI components.
**Conformance target:** WCAG 2.2 AA.
**Status:** All **P0 (critical)** findings fixed on 2026-07-04. P1–P3 remain open.

---

## 1. Summary & methodology

Every route under `src/app/**` and every component under `src/components/**` was reviewed for
accessibility gaps, with the highest-impact findings spot-verified against source
(`company-tabs.tsx`, `companies-table.tsx`, `authenticated-shell.tsx`, `layout.tsx`).

The shared primitives in `src/components/ui/*` are built on **base-ui** (not Radix). base-ui
provides solid built-in accessibility — focus trapping, focus restoration, roving `tabindex`,
`aria-modal`, and the correct roles — **but only when those primitives are actually used.** The
gaps found cluster in three categories:

1. **Hand-rolled interactive widgets** that bypass the accessible primitives (tabs, combobox,
   clickable rows).
2. **Icon-only controls** rendered without an accessible name.
3. **Color-only** encoding of information (profit/loss, allocation status).

Counts: **3 critical** (✅ all fixed), **5 serious**, **6 moderate**, **3 minor/verify**, plus
**4 WCAG-2.2 criteria** requiring a runtime pass to confirm.

The `lang="en"` attribute (`app/layout.tsx:29`) and the `<main>` landmark
(`authenticated-shell.tsx:50`) are present — good baselines to build on.

## 3. Detailed findings

### P0 — Critical (keyboard / operability blockers) — ✅ ALL FIXED (2026-07-04)

**1. Clickable table rows are not keyboard-operable** — `dashboard/companies-table.tsx:707`
*WCAG 2.1.1 Keyboard (A), 4.1.2 Name/Role/Value (A).*
✅ **Fixed:** the company-name cell now contains a real focusable
`<Link href="/company/{id}">` with a `focus-visible` ring. Keyboard and AT users can Tab to it and
press Enter; the row-level `onClick` is retained as a mouse convenience (the link
`stopPropagation`s to avoid a double navigation). This also gives middle-click / "open in new tab"
for free.

**2. Company detail tabs are hand-rolled, not a real tab widget** — `company/company-tabs.tsx`
*WCAG 4.1.2 (A), 2.1.1 (A).*
✅ **Fixed:** migrated from `<button>`-toggled `hidden` divs to the base-ui `Tabs` primitive
(`Tabs.Root` / `Tabs.List` / `Tabs.Tab` / `Tabs.Panel`), which supplies
`role="tablist"/"tab"/"tabpanel"`, `aria-selected`, `aria-controls`/`aria-labelledby`, and
Left/Right/Home/End arrow-key navigation. Panels use `keepMounted` to preserve the previous
always-mounted behaviour (tab-local form state survives switching). The underline styling is
preserved via `data-active:` variants, and a `focus-visible` ring was added.

**3. Stock search is a hand-rolled combobox without ARIA** — `company/stock-search.tsx`
*WCAG 4.1.2 (A), 2.1.1 (A).*
✅ **Fixed:** the input now exposes `role="combobox"`, `aria-expanded`, `aria-controls`,
`aria-autocomplete="list"`, and `aria-activedescendant`; the results list is `role="listbox"` with
`role="option"` + `aria-selected` items. Added full keyboard navigation (ArrowUp/Down, Home/End,
Enter to select, Escape to close) with the active option scrolled into view, following the
focus-stays-in-input activedescendant pattern.

### P1 — Serious

**4. Icon-only controls with no accessible name** — each needs an `aria-label`.
*WCAG 4.1.2 (A), 2.4.4 (A).*
- `auth/user-nav.tsx:25` — avatar/initials menu trigger.
- `import/page.tsx:267` — remove-file `XCircle` button; `:411` — delete-history `Trash2` button.
- `account/accounts-manager.tsx:151,160` — `Pencil` / `Trash2` per account.
- `portfolio/create-portfolio-dialog.tsx:106` — 8 color-swatch buttons.
- `ui/rich-text-editor-impl.tsx` — ColorPicker swatches, hidden image/PDF file inputs.
- `dashboard/delete-company-dialogs.tsx` — trash trigger.
- `ui/select.tsx:157,176` — scroll up/down buttons.

**5. Rich-text toolbar uses `title` instead of `aria-label`** —
`ui/rich-text-editor-impl.tsx:138-168,343-532`
`ToolbarButton`, `FontSizeSelect`, and `TableMenu` rely on `title`, which is not reliably announced
by screen readers. *WCAG 4.1.2 (A).*
**Fix:** add `aria-label` to each control, give the toolbar `role="toolbar"`, and add
`aria-disabled` to disabled buttons.

**6. Color-only encoding of financial status** — meaning conveyed by red/green alone.
*WCAG 1.4.1 Use of Color (A).*
- `dashboard/allocation-summary-bar.tsx:56-87` — under / in-range / over by background color only.
- `dashboard/portfolio-pnl-bar.tsx:55-62` and `companies-table.tsx:748-794` — P&L profit/loss.
- `companies-table.tsx` — allocation border colors.
**Fix:** always render an explicit `+`/`-` sign or a text/`aria-label` status
("Over-allocated", "Profit ₹5,000").

**7. Data tables missing `scope` on header cells** — `ui/table.tsx` `TableHead` (the base for all
tables), plus `company/holdings-tab.tsx:195`, `company/projection-grid.tsx:88` (which also needs
`scope="row"` on row-label cells), and `company/valuation-scenarios.tsx`.
*WCAG 1.3.1 Info & Relationships (A).* **Fix:** add `scope="col"`/`"row"`.

**8. Star ratings have no text alternative** — `dashboard/companies-table.tsx:723`
(`"★".repeat(n)`) and elsewhere. Screen readers read a run of star glyphs with no rating context.
*WCAG 1.1.1 (A) / 1.3.1 (A).* **Fix:** wrap with `aria-label="{n} of 5 stars"` and `aria-hidden`
the glyphs.

### P2 — Moderate

**9. No skip-to-content link** — `app/layout.tsx` / `authenticated-shell.tsx:50`. A `<main>` exists
but has no `id`, and there is no skip link to bypass the header nav. *WCAG 2.4.1 Bypass Blocks (A).*
**Fix:** add a visually-hidden skip link + `id="main-content"` on `<main>`.

**10. Navigation missing current-page indication** — `layout/app-header.tsx:28-38`. Active link is
styled visually only. *WCAG 4.1.2 (A).* **Fix:** add `aria-current="page"` from `pathname`.

**11. Async status not announced** — no `role="status"` / `aria-live="polite"`.
*WCAG 4.1.3 Status Messages (AA).*
- Dashboard "Loading companies…" `app/(authenticated)/page.tsx:67`.
- Import "Importing…" progress `import/page.tsx:325`.
- P&L / allocation bars update dynamically with no live region.
- Sonner toasts — verify the library emits `aria-live` (it generally does).

**12. Form-labelling gaps** — *WCAG 1.3.1 (A), 3.3.2 Labels or Instructions (A), 4.1.2 (A).*
- `company/company-form.tsx:191` — StockSearch wrapped in `<Label>` with no `htmlFor`/input `id`.
- `account/account-select.tsx:30` — native `<select>` with no associated label.
- `settings/allocation-ranges-editor.tsx:61-79` — min/max % inputs labelled only by a `★` glyph;
  add `aria-label`.
- `company/holdings-tab.tsx:287` — add-holding input group lacks `<fieldset>`/`<legend>`.
- Verify `ui/input.tsx` + `ui/label.tsx` forward `id`/`htmlFor` (edit-company-dialog depends on it).

**13. Per-page titles not set** — only the root title exists (`app/layout.tsx:16`). Login,
dashboard, import, new-company, and company-detail pages define no `metadata` /
`generateMetadata`, so the browser tab / SR page title never changes on navigation.
*WCAG 2.4.2 Page Titled (A).*

**14. Card titles are `<div>`, not headings** — `ui/card.tsx:36`. This breaks the heading outline
that screen-reader users navigate by. *WCAG 1.3.1 (A).* **Fix:** allow a semantic `as="h2/h3"`
and audit overall heading order (single `h1` per page).

### P3 — Minor / verify

**15. Decorative elements not hidden from AT** — `ui/tooltip.tsx:59` Arrow, `ui/separator.tsx`.
Add `aria-hidden="true"` (base-ui may already; verify). *WCAG 1.3.1 (A).*

**16. `aria-modal` + focus management on dialogs** — `ui/dialog.tsx`, `ui/alert-dialog.tsx`,
`ui/sheet.tsx`. base-ui almost certainly sets `aria-modal` and manages focus trap/restore;
**verify at runtime** rather than assume. *WCAG 4.1.2 (A), 2.4.3 Focus Order (A).*

**17. `ui/skeleton.tsx`** — loading placeholder with no `role="status"`/`aria-busy`. Prefer a
single live-region wrapper over per-skeleton announcements. *WCAG 4.1.3 (AA).*

## 4. Quick wins (high impact, low effort)

1. Add `aria-label` to every icon-only button (finding 4).
2. Add `scope="col"`/`"row"` to all `<th>` — fix once in `ui/table.tsx` `TableHead` (finding 7).
3. Always render a `+`/`-` sign (or text) on P&L and allocation status (finding 6).
4. Replace the hand-rolled tabs in `company-tabs.tsx` with the existing base-ui `ui/tabs.tsx`
   (finding 2).
5. Add a skip-to-content link + `id` on `<main>` (finding 9).
6. Add `aria-current="page"` to header nav links (finding 10).

## 5. Requires manual / runtime verification

These cannot be confirmed by static reading alone and warrant an axe/Lighthouse pass plus a
keyboard/screen-reader walkthrough:

- **Findings 15–17** — decorative `aria-hidden`, dialog `aria-modal`/focus trap, skeleton status.
- **2.4.7 Focus Visible (AA) / 2.4.11 Focus Not Obscured (AA, new in 2.2) / 2.4.13 Focus Appearance
  (AAA).** Grep for `outline-none` / `focus:outline-none` without a `focus-visible` ring; confirm
  the sticky header does not obscure focused elements on scroll.
- **2.5.8 Target Size (Minimum) 24×24 px (AA, new in 2.2).** The icon buttons and the 8 color
  swatches in create-portfolio / rich-text ColorPicker are likely under 24 px — measure.
- **1.4.3 Contrast (Minimum) (AA).** `--muted-foreground` (used pervasively) and
  `bg-destructive/10` + `text-destructive` in dark mode are both suspect — run a contrast checker.
- **3.3.8 Accessible Authentication (AA, new in 2.2).** Google-OAuth-only login has no cognitive
  puzzle, so it likely passes; confirm the login control is a real labelled `<button>`/`<a>` with
  text.
